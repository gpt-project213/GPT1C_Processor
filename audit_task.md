11. Правила проведения аудита

11.1. Общие правила

Не делать выводов без чтения файла

Если проблема не доказана кодом — не заявлять её как факт

Не выдавать общий обзор за полный аудит

11.3. Правила глубины

Читать каждый скрипт

Не вносить правок

Не переписывать код

Раздел ТЗ: Завершение аудита и фиксация результата

После завершения полного анализа проекта исполнитель обязан выполнить контрольную процедуру проверки и сформировать итоговый отчёт.

1. Компиляционная проверка Python-файлов

Для каждого Python-файла проекта должна быть выполнена проверка синтаксической корректности.

Проверка выполняется командой:

python -m py\_compile <путь\_к\_файлу>

Проверка проводится для всех Python-файлов проекта, независимо от того, были ли обнаружены проблемы.

Цель проверки — выявить:

синтаксические ошибки

повреждённые файлы

несовместимость версий Python

скрытые ошибки импорта

2. Синхронизация проекта с репозиторием

После завершения аудита необходимо выполнить проверку состояния репозитория и синхронизацию проекта.

Для этого запускается скрипт:

tools/sync\_project\_to\_github.ps1

Скрипт выполняется без внесения изменений в код проекта и используется только для:

проверки состояния git-репозитория

фиксации текущего состояния проекта

синхронизации с GitHub

3. Итоговый отчёт аудита

После выполнения всех этапов аудита необходимо вывести итоговый отчёт.
